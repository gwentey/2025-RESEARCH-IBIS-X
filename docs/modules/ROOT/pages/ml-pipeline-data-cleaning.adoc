:page-layout: docs

== Vue d'ensemble

L'étape 3 du ML Pipeline permet d'analyser et de nettoyer automatiquement vos données avant l'entraînement du modèle. Cette fonctionnalité détecte les valeurs manquantes, propose des stratégies de nettoyage personnalisées par colonne et applique les corrections de manière intelligente.

== Fonctionnalités principales

=== Analyse automatique des colonnes

Lorsque vous cliquez sur "Lancer l'Analyse Intelligente", le système :

* Examine chaque colonne individuellement
* Identifie le type de données (numérique, catégoriel, temporel)
* Calcule le pourcentage de valeurs manquantes
* Attribue un score de qualité global au dataset
* Recommande des stratégies de nettoyage adaptées

=== Tableau de configuration multi-colonnes

Le tableau d'analyse affiche pour chaque colonne :

[cols="1,3"]
|===
|Élément |Description

|**Nom de la colonne**
|Nom avec icône indiquant le type de données

|**Type**
|Type de données détecté (Integer, Float, String, DateTime)

|**Pourcentage manquant**
|Barre visuelle et pourcentage des valeurs manquantes

|**Stratégie de nettoyage**
|Menu déroulant avec options adaptées au type de données

|**Paramètres**
|Configuration spécifique selon la stratégie choisie

|**Aperçu**
|Bouton pour visualiser l'effet avant d'appliquer
|===

== Stratégies de nettoyage disponibles

=== Suppression
* **Supprimer colonne** : Retire entièrement la colonne (désactivé pour la variable cible)
* **Supprimer lignes** : Retire les lignes contenant des valeurs manquantes

=== Imputation simple
* **Moyenne** : Remplace par la moyenne (données numériques)
* **Médiane** : Remplace par la médiane (données numériques)
* **Mode** : Remplace par la valeur la plus fréquente
* **Constante** : Remplace par une valeur fixe définie

=== Imputation avancée
* **KNN** : Utilise les K plus proches voisins
* **MICE** : Multiple Imputation by Chained Equations
* **Random Forest** : Prédiction basée sur un modèle (données numériques)

=== Interpolation (données temporelles)
* **Linéaire** : Interpolation linéaire entre les points
* **Spline** : Interpolation par spline cubique
* **Forward Fill** : Propagation de la dernière valeur connue
* **Backward Fill** : Propagation inverse de la prochaine valeur

== Utilisation étape par étape

. **Accéder à l'étape 3** : Après avoir sélectionné votre dataset et configuré l'objectif
. **Lancer l'analyse** : Cliquez sur "Lancer l'Analyse Intelligente"
. **Examiner les résultats** : 
   - Score de qualité global
   - Nombre de colonnes nécessitant un nettoyage
   - Détails par colonne dans le tableau
. **Configurer les stratégies** :
   - Utilisez "Configuration Intelligente" pour appliquer les recommandations
   - Ou ajustez manuellement chaque colonne
. **Prévisualiser** : Cliquez sur "Aperçu" pour voir l'effet sur un échantillon
. **Valider** : Cliquez sur "Valider la Configuration" pour appliquer

== Recommandations et bonnes pratiques

[TIP]
====
* Pour moins de 15% de valeurs manquantes : privilégiez l'imputation
* Pour plus de 70% de valeurs manquantes : considérez la suppression de la colonne
* Pour les données temporelles : utilisez l'interpolation
* La variable cible ne peut pas être supprimée
====

== Fonctionnalités avancées

=== Fusion de datasets
Possibilité de croiser plusieurs datasets en définissant des clés de jointure (INNER, LEFT, RIGHT, OUTER JOIN).

=== Export Python
Génération du code Python équivalent pour reproduire le pipeline de nettoyage en dehors de la plateforme.

== Résolution de problèmes

.Le tableau ne s'affiche pas après l'analyse
[WARNING]
====
Si le tableau n'apparaît pas :

. Vérifiez que l'analyse est terminée (pas de message "Analyse en cours")
. Consultez la console du navigateur pour d'éventuelles erreurs
. Rafraîchissez la page et relancez l'analyse
====

.Stratégies non disponibles
[NOTE]
====
Certaines stratégies ne sont disponibles que pour des types de données spécifiques :
* Moyenne/Médiane : uniquement pour les données numériques
* Interpolation : uniquement pour les séries temporelles
====
