= Gestion des secrets
:description: Inventaire, stockage, rotation

== Principes

. *Jamais* de secret en clair dans Git.
. *Jamais* de valeur sensible hardcodée dans le code (Pydantic `BaseSettings` obligatoire).
. *Rotation* au minimum lors d'un changement d'équipe ou d'une suspicion de fuite.

== Inventaire des secrets

[cols="1,1,3"]
|===
| Nom | Portée | Contenu

| `JWT_SECRET_KEY` | api-gateway | Clé HS256, 32 octets aléatoires minimum.
| `DATABASE_URL` | tous services | `postgresql://user:password@host:5432/ibis_x_db`
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | api-gateway | Credentials OAuth Google.
| `OAUTH_REDIRECT_URL` | api-gateway | URL de callback (whitelist Google Cloud).
| `KAGGLE_USERNAME` / `KAGGLE_KEY` | job import Kaggle | API Kaggle.
| `OPENAI_API_KEY` | ml-pipeline-service, xai-engine-service | Appels OpenAI.
| `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` | tous services (local) | MinIO locale.
| `AZURE_STORAGE_CONNECTION_STRING` | tous services (prod) | Azure Blob.
|===

== Stockage

=== En local (Minikube)

* `.env` à la racine du dépôt (gitignored) — variables d'environnement pour le développeur.
* Script `update-local-secrets.py` synchronise `.env` → `k8s/overlays/minikube/secrets.yaml` (généré, non commité).
* Kubernetes Secrets injectés dans les Deployments via `envFrom` / `env`.

=== En production (Azure AKS)

* Provisioning via Terraform : création d'un Azure Key Vault.
* Options (au choix selon maturité) :
** *Option A (PoC)* — Secrets K8s renseignés manuellement via `kubectl apply -f secrets.yaml` (manifest généré depuis le Key Vault, jamais commité).
** *Option B (cible)* — External Secrets Operator (ESO) synchronise automatiquement Key Vault → Secrets K8s.

== Rotation

[cols="1,2,2"]
|===
| Secret | Fréquence recommandée | Procédure

| `JWT_SECRET_KEY` | Annuellement ou sur incident | Invalide tous les JWT en cours ; utilisateurs devront se reconnecter. Runbook : xref:04-operationnel/runbooks/rotation-secrets.adoc[Rotation de secrets].
| `OPENAI_API_KEY` | Semestrielle ou sur incident | Générer nouvelle clé dans la console OpenAI, mettre à jour Key Vault, rollout du Deployment.
| `KAGGLE_KEY` | Annuellement | Régénérer via Kaggle.
| `DATABASE_URL` (password) | Semestrielle | Rotation via `ALTER USER` PostgreSQL + mise à jour secret + rollout.
| `GOOGLE_CLIENT_SECRET` | Sur incident uniquement | Via Google Cloud Console.
|===

== Détection de fuite

* Pre-commit hook (à ajouter) : `gitleaks` ou `detect-secrets`.
* GitHub secret scanning activé sur le dépôt.
* Revue manuelle des PRs (mainteneur).

== Bonnes pratiques développeur

. Utiliser `.env.example` (committé) comme référence — **ne jamais** committer `.env`.
. Ne pas logger `settings.dict()` en prod.
. Pour tester en local, valeurs factices suffisantes ; ne jamais réutiliser des valeurs de prod.
. Si un secret est committé par erreur : révoquer immédiatement + rotation + `git rebase` + force-push coordonné avec les mainteneurs.
