= Politique de sécurité IBIS-X
:description: Principes, périmètre, responsabilités

== Statut

IBIS-X est un **Proof of Concept académique** (Master 2 MIAGE) destiné à la démonstration de concepts XAI. Il n'est pas conçu pour traiter des données sensibles réelles. La présente politique documente toutefois les dispositions prises.

== Principes directeurs

. *Defense in depth* — plusieurs couches de contrôle (JWT, Kubernetes Secrets, TLS, isolation namespace).
. *Least privilege* — rôles utilisateurs minimaux (`user` par défaut, `admin` restreint).
. *Secrets never in Git* — jamais de valeurs sensibles commitées ; variables d'environnement + Kubernetes Secrets.
. *Dépendances à jour* — surveillance régulière des CVE (GitHub Dependabot activé).
. *Conteneurs non-root* — toutes les images Docker `ibis-x-*` s'exécutent avec un utilisateur non privilégié.

== Périmètre couvert

* Code applicatif : 4 services backend Python + frontend Angular.
* Infrastructure : Kubernetes manifests (`k8s/`), Terraform (`terraform/`), GitHub Actions.
* Dépendances directes (`requirements.txt`, `package.json`).

== Hors périmètre

* Sécurité physique des hôtes (Minikube local / AKS Azure).
* Sécurité des services managés Azure (Azure Key Vault, Azure Blob Storage — la responsabilité est Microsoft sous SLA).
* Attaques nécessitant un accès admin OS préalable.

== Mesures en place

[cols="1,3"]
|===
| Catégorie | Mesures

| Authentification
| JWT HS256 via fastapi-users. OAuth Google. Mots de passe bcrypt. Tokens courte durée (1h).

| Autorisation
| Rôles `user` / `contributor` / `admin`. `is_superuser` propriété calculée (`role == 'admin'`).

| Transport
| HTTPS en prod via Ingress Azure + cert-manager. HTTP local seul.

| Secrets
| Kubernetes Secrets en local ; Azure Key Vault (+ External Secrets Operator envisagé) en prod. `JWT_SECRET_KEY` ≥ 32 octets aléatoires.

| Isolation
| Namespace K8s unique `ibis-x`. NetworkPolicies non déployées en PoC (évolution possible).

| Dépendances
| Versions pinnées dans `requirements.txt` / `package-lock.json`. Dependabot GitHub sur CVE critiques.

| Conteneurs
| Images multi-stage, base slim Python / nginx, utilisateur non-root.

| CORS
| Restrictif en prod (origine frontend uniquement). Permissif en dev (`localhost:*`).

| Signalement
| Voir `SECURITY.md` à la racine du dépôt.
|===

== Responsabilités

[cols="1,2"]
|===
| Acteur | Responsabilité

| Mainteneur du dépôt | Revue des PR, mise à jour des dépendances, arbitrage des rapports de vulnérabilité.
| Développeur contributeur | Respect des conventions (pas de hardcoding, pas de secret en Git, `Pydantic BaseSettings`).
| Utilisateur final (testeur PoC) | Ne pas uploader de données personnelles réelles.
|===
