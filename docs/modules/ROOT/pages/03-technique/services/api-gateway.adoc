= Service — api-gateway
:description: Authentification, autorisation, routage reverse-proxy
:toc: left

== Rôle

Point d'entrée unique de la plateforme :

* Authentification JWT + OAuth Google (via https://fastapi-users.github.io[fastapi-users]).
* Autorisation par rôle (`user`, `contributor`, `admin`).
* Reverse-proxy vers les services métier (`service-selection`, `ml-pipeline-service`, `xai-engine-service`).
* Propagation du `user_id` authentifié aux services downstream.

== Périmètre

✅ **Inclut** : login/logout/register, OAuth callback, gestion `User.role`, refresh token, CORS, rate-limiting (si activé).

❌ **Exclut** : logique métier datasets, ML, XAI — jamais de SQL sur ces domaines ici.

== Modèle `User`

[cols="1,1,2"]
|===
| Champ | Type | Remarque

| `id` | UUID | Clé primaire
| `email` | string (unique) | Identifiant principal
| `hashed_password` | string | bcrypt
| `role` | string | `user` (défaut) / `contributor` / `admin`
| `is_active` | bool | Soft-disable
| `is_superuser` | *propriété calculée* | `True` ssi `role == 'admin'` — **pas une colonne**
| `oauth_accounts` | table associée | Google OAuth (fastapi-users schema)
|===

Migration pivot : `sync_role_is_superuser_permanent` (gateway) — garantit la cohérence entre `role` et `is_superuser` après suppression du champ colonne.

== Endpoints principaux

[cols="1,1,3"]
|===
| Méthode | Route | Rôle

| POST | `/auth/jwt/login` | Login classique → JWT
| POST | `/auth/jwt/logout` | Invalidation token
| POST | `/auth/register` | Création utilisateur (rôle `user`)
| GET  | `/auth/google/authorize` | Démarrer OAuth Google
| GET  | `/auth/google/callback` | Callback OAuth
| GET  | `/users/me` | Profil courant
| PATCH | `/users/me` | Mise à jour profil
| DELETE | `/users/me` | Suppression compte (droit RGPD)
| *toutes les routes `/api/*`* | reverse-proxy | délègue à service-selection / ml-pipeline / xai-engine
|===

== Dépendances

* PostgreSQL (`ibis_x_db`) — `alembic_version_gateway`.
* Google OAuth (variables `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`).
* Variables sensibles : `JWT_SECRET_KEY`, `OAUTH_REDIRECT_URL`.

== Sécurité

Voir xref:05-audit/securite/politique.adoc[Politique de sécurité] pour le modèle global.

* JWT HS256 signé avec `JWT_SECRET_KEY` (32 octets minimum).
* Mots de passe hashés bcrypt (coût par défaut de fastapi-users).
* Tokens à durée limitée (1h par défaut).
* CORS restrictif en prod (origine frontend uniquement).
