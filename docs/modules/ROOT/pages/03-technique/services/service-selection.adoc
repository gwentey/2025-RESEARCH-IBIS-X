= Service — service-selection
:description: Catalogue de datasets, filtres, upload, multi-fichiers
:toc: left

== Rôle

Gestion du cycle de vie des datasets :

* Catalogue filtrable (critères techniques, éthiques, métier).
* Upload avec manifest et détection automatique des colonnes.
* Accès multi-fichiers (un dataset peut contenir plusieurs fichiers).
* Import Kaggle via job Kubernetes dédié.

== Modèle de données

Trois tables principales — voir xref:03-technique/architecture/modele-donnees.adoc[ERD] :

* *`datasets`* : métadonnées globales, `storage_path` (préfixe stockage objets), scores (technique/éthique/métier), lien vers uploadeur.
* *`dataset_files`* : un dataset peut avoir plusieurs fichiers (train.csv, test.csv…) avec chacun son chemin, type, nombre de lignes.
* *`file_columns`* : colonnes de chaque fichier avec type, statistiques (min/max/mean/nulls), flag PII.

Autres tables associées : `dataset_relationships` (liens entre datasets), `ethical_templates` (grilles d'évaluation).

== Endpoints principaux

[cols="1,1,3"]
|===
| Méthode | Route | Rôle

| GET | `/datasets` | Listing paginé + filtres (query params)
| GET | `/datasets/{id}` | Détail complet (fichiers, colonnes, scores, métadonnées enrichies)
| POST | `/datasets` | Upload nouveau dataset (multipart + manifest)
| PUT | `/datasets/{id}` | Mise à jour métadonnées
| DELETE | `/datasets/{id}` | Suppression (dataset + fichiers en storage)
| GET | `/datasets/{id}/preview` | Échantillon de lignes (limit/offset)
| GET | `/datasets/{id}/columns` | Liste des colonnes avec statistiques
| GET | `/ethical-templates` | Templates d'évaluation éthique
|===

== Stockage

Préfixe : `ibis-x-datasets/{dataset_id}/`.

* *Local* : MinIO (bucket `ibis-x-datasets`).
* *Prod* : Azure Blob Storage (container `ibis-x-datasets`).

Basculement contrôlé par la variable `STORAGE_BACKEND=minio|azure`.

== Import Kaggle

Workflow dédié via Job Kubernetes :

* Configuration : `datasets/kaggle-import/kaggle_datasets_config.yaml`.
* Credentials : secret K8s `kaggle-secrets`.
* Conversion CSV → Parquet (Snappy) pour optimisation (gains 10-50x).
* Cache local 7 jours pour éviter les re-téléchargements.

Voir `datasets/kaggle-import/README.md` pour le détail opérationnel.

== Analyse qualité IA

L'endpoint `POST /datasets/{id}/analyze` délègue à `ml-pipeline-service` (qui publie sur `ai_queue`). Le résultat est caché dans `data_quality_analyses` et exposé via `GET /datasets/{id}/analysis`.

Cette frontière est importante : `service-selection` ne contient pas de logique OpenAI ; il délègue.

== Dépendances

* PostgreSQL (`ibis_x_db`) — `alembic_version_selection`.
* MinIO / Azure Blob.
* `ml-pipeline-service` pour déclencher les analyses IA (via HTTP interne ou Redis broker).
