= Modèle de données (ERD)
:description: Schéma PostgreSQL — tables principales et relations

Base unique *`ibis_x_db`* partagée par les 4 services. Chaque service gère ses propres migrations via une `version_table` Alembic dédiée : `alembic_version_gateway`, `alembic_version_selection`, `alembic_version_ml_pipeline`, `alembic_version_xai`.

== Invariants du schéma

* *`artifact_uri`* est la colonne des expériences pointant vers le modèle sérialisé (renommage depuis `model_uri` via migration `002_rename_model_to_artifact`).
* *Dataset multi-fichiers* : un dataset peut contenir plusieurs fichiers (train/test/validation). L'accès se fait via `storage_path` (préfixe) + table `dataset_files` + `file_columns`.
* *`data_quality_analyses`* est un **cache** des analyses IA OpenAI pour éviter de recalculer à chaque consultation (clé d'invalidation : `dataset_version`).
* *`users.is_superuser`* n'est **pas** une colonne : c'est une **propriété calculée** (`role == 'admin'`).
* *Chat XAI* : limité à 5 messages utilisateur par `explanation_request` (contrôle applicatif, pas SQL).
