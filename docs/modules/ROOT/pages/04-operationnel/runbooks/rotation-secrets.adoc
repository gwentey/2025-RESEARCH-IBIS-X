= Runbook — Rotation de secrets
:description: Remplacer un secret compromis ou arrivé à échéance

== Inventaire

Voir xref:05-audit/securite/gestion-secrets.adoc[Gestion des secrets] pour la liste complète.

== Procédure générale (8 étapes)

. *Générer la nouvelle valeur* (Azure Key Vault, Google Cloud Console, OpenAI console…).
. *Stocker* dans Azure Key Vault en prod (ou mettre à jour `.env` en local).
. *Mettre à jour* le Secret Kubernetes :
+
[source,bash]
----
# Local
python scripts/production/update-local-secrets.py

# Prod (si ESO non activé)
kubectl create secret generic {secret-name} \
  --from-literal=KEY="$NEW_VALUE" \
  --dry-run=client -o yaml \
  | kubectl apply -f -
----
. *Rollout* des Deployments consommant le secret :
+
[source,bash]
----
kubectl rollout restart deployment/{service}-deployment -n ibis-x
----
. *Vérifier* santé post-rollout :
+
[source,bash]
----
kubectl get pods -n ibis-x
kubectl logs -f deployment/{service}-deployment -n ibis-x
----
. *Tester* le parcours critique (login si JWT, upload si Blob, training si OpenAI).
. *Révoquer* l'ancienne valeur côté fournisseur (Google, OpenAI, Kaggle).
. *Documenter* dans `CHANGELOG.md` sous `[Unreleased] / Security`.

== Cas particuliers

=== JWT_SECRET_KEY

Rotation = invalidation de tous les tokens en circulation. Les utilisateurs devront se reconnecter. **Ne pas faire en heure de pointe**.

=== DATABASE_URL (password PostgreSQL)

. `ALTER USER ibis_x_user WITH PASSWORD 'NEW_PWD';`
. Mettre à jour le Secret K8s.
. Rollout des 4 services + 2 workers **simultanément** (sinon certains pods vont se déconnecter).

=== OPENAI_API_KEY

. Créer une nouvelle clé dans console OpenAI.
. Mettre à jour Secret K8s.
. Rollout `ml-pipeline-service`, `ml-pipeline-worker`, `xai-engine-service`, `xai-engine-worker`.
. Tester `analyze_dataset_with_ai` + chat XAI.
. Révoquer ancienne clé.

=== GOOGLE_CLIENT_SECRET

. Générer nouveau secret dans Google Cloud Console.
. Ajouter au Secret K8s (garder l'ancien provisoirement pour les sessions OAuth en cours).
. Rollout `api-gateway`.
. Attendre 1h (expiration sessions).
. Retirer l'ancien secret.

== Check-list post-rotation

* [ ] Secret K8s mis à jour et rollé.
* [ ] Tous les pods consommateurs en `Running` + `Ready`.
* [ ] Parcours critique testé (login, upload, training, XAI).
* [ ] Ancienne valeur révoquée côté fournisseur.
* [ ] `CHANGELOG.md` mis à jour.
* [ ] Si violation : suivre xref:04-operationnel/runbooks/incident-response.adoc[Incident response].
