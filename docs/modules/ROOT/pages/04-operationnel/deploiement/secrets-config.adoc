= Secrets & configuration
:description: Orchestration des ConfigMaps et Secrets K8s

== Séparation

* **ConfigMap** → configuration non sensible (URLs internes, flags, paramètres de comportement).
* **Secret** → valeurs sensibles (credentials, clés privées, tokens).

Jamais l'inverse. Jamais de valeur hardcodée dans un manifest.

== Sources

[cols="1,2,2"]
|===
| Env | Source de vérité | Mécanisme d'injection

| *Local (Minikube)* | `.env` (gitignoré) | `scripts/production/update-local-secrets.py` génère `k8s/overlays/minikube/secrets.yaml`
| *Prod (AKS)* | Azure Key Vault | Option A : `kubectl apply` manuel post-provisionning. Option B (cible) : External Secrets Operator sync automatique.
|===

== Consommation dans les Deployments

Via `envFrom` (bloc entier) ou `env` (variables individuelles) :

[source,yaml]
----
spec:
  template:
    spec:
      containers:
        - name: service
          envFrom:
            - configMapRef:
                name: service-config
            - secretRef:
                name: jwt-secrets
          env:
            - name: POD_NAMESPACE
              valueFrom:
                fieldRef:
                  fieldPath: metadata.namespace
----

== Variables par service (exemples)

[cols="1,3"]
|===
| Service | Variables principales

| api-gateway
| `JWT_SECRET_KEY`, `DATABASE_URL`, `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `OAUTH_REDIRECT_URL`, `FRONTEND_URL`

| service-selection
| `DATABASE_URL`, `MINIO_*` ou `AZURE_STORAGE_CONNECTION_STRING`, URL interne `ML_PIPELINE_SERVICE_URL`

| ml-pipeline-service
| `DATABASE_URL`, `REDIS_URL`, `MINIO_*`/`AZURE_*`, `OPENAI_API_KEY`

| xai-engine-service
| `DATABASE_URL`, `REDIS_URL`, `MINIO_*`/`AZURE_*`, `OPENAI_API_KEY`

| frontend (nginx)
| Aucune sensible ; `API_BASE_URL` via ConfigMap
|===

== Rollout après mise à jour

Après modification d'un Secret ou ConfigMap :

[source,bash]
----
kubectl rollout restart deployment/{service}-deployment -n ibis-x
----

Les pods redémarrent avec les nouvelles valeurs. Sinon les anciennes valeurs restent en mémoire (envFrom évalué au démarrage).

== Règles impératives

. **Jamais** de secret en clair dans un manifest Kustomize versionné.
. **Jamais** `secrets.yaml` commité — toujours `.gitignore`.
. **Toujours** utiliser `Pydantic BaseSettings` côté code (validation + typage des variables).
. **Toujours** valider une valeur avant démarrage (le pod doit fail-fast si variable manquante).
