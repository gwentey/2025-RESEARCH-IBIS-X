== Concepts domaine

*Dataset* — jeu de données tabulaire (CSV/Parquet/JSON). Peut être multi-fichiers. Référencé par `storage_path`, `dataset_files`, `file_columns`.

*Experiment* — exécution d'un pipeline ML avec une config précise. Produit un artefact référencé par `artifact_uri`.

*Artifact* — modèle sérialisé (joblib) dans le stockage objets. `artifact_uri` remplace l'ancien `model_uri`.

*Explanation Request* — tâche async XAI (SHAP / LIME / LLM) routée sur `xai_queue`.

*Chat* — conversation utilisateur ↔ LLM autour d'une explication. 5 questions max par session.

*Score (éthique / technique / métier)* — évaluation 0-10 d'un dataset sur 3 dimensions, affichée dans le catalogue.

*Manifest* — fichier accompagnant un upload, décrit structure + métadonnées éthiques.

*DataQualityAnalysis* — cache des analyses IA pour éviter des appels OpenAI répétés. Invalidé par `dataset_version`.

== Concepts techniques projet

*`ibis_x_db`* — BDD unique partagée. Isolation des migrations via `version_table` Alembic par service.

*`version_table`* — table Alembic par service : `alembic_version_gateway`, `alembic_version_selection`, `alembic_version_ml_pipeline`, `alembic_version_xai`.

*Queue Celery* — 3 actives dans IBIS-X : `ml_queue`, `ai_queue`, `xai_queue`. **Pas de `llm_queue`.**

*Port-forward Skaffold* — redirection locale 8080/9000. **Ne pas confondre** avec ports internes (8088, 8081, 8082, 8083).

*Overlay Kustomize* — surcouche par environnement (`minikube`, `azure`).
