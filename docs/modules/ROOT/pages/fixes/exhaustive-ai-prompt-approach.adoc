== Vue d'ensemble

Ce document décrit l'approche exhaustive et pédagogique pour que l'IA OpenAI comprenne par elle-même que les colonnes numériques nécessitent de la régression, sans aucun court-circuit programmatique.

=== Philosophie

L'IA doit être **éduquée** et **guidée** pour comprendre les règles fondamentales du Machine Learning, pas contournée. Une vraie compréhension permet des réponses plus nuancées et pédagogiques.

== Problème antérieur

=== Tentatives précédentes échouées

. **Prompt basique** : L'IA ignorait les règles simples
. **Court-circuit programmatique** : Pas de vraie réponse IA, juste du code
. **Rules insuffisantes** : Manque de clarté et d'exemples

=== Symptômes persistants

* IA recommande "Classification" pour `Addiction_Level` (numerical)
* Ignore le type de données et se fie au nom
* Explications génériques, pas spécifiques au dataset

== Solution exhaustive implémentée

=== 1. System Prompt Renforcé

.Règles fondamentales claires
[source,text]
----
🎯 RÈGLE FONDAMENTALE ABSOLUE :

LE TYPE DE DONNÉES DE LA COLONNE CIBLE DÉTERMINE TOUT :

✅ TYPES NUMÉRIQUES = RÉGRESSION OBLIGATOIRE
- numerical, numeric, float, float64, double, decimal
- PEU IMPORTE le nom de la colonne !
- Exemples corrects: "Addiction_Level" (numerical) → RÉGRESSION

❌ NE PAS SE FIER AU NOM DE LA COLONNE !
----

=== 2. Prompt Principal Exhaustif

.Structure pédagogique étape par étape
[source,text]
----
🔍 ANALYSE ÉTAPE PAR ÉTAPE REQUISE :

Étape 1 - ANALYSEZ LE TYPE DE LA COLONNE CIBLE
Étape 2 - APPLIQUEZ LA RÈGLE
Étape 3 - JUSTIFIEZ VOTRE CHOIX

⚠️ ATTENTION AUX PIÈGES COURANTS :
- "Addiction_Level" de type numerical → RÉGRESSION (pas classification)

CONTRE-EXEMPLES À ÉVITER :
❌ MAL : "Addiction_Level semble être une catégorie"
✅ BIEN : "Addiction_Level est de type numerical → RÉGRESSION"
----

=== 3. Exemples Concrets et Spécifiques

Le prompt inclut des exemples **explicites** du problème rencontré :

* `Addiction_Level` type="numerical" → RÉGRESSION
* `Customer_Satisfaction` type="float" → RÉGRESSION  
* `Risk_Score` type="numeric" → RÉGRESSION

=== 4. Méthode d'Analyse Forcée

L'IA doit suivre une méthode rigoureuse :

. **REGARDER** le type de données (pas le nom !)
. **SI numérique** → RÉGRESSION (même si nom suggère catégorie)
. **SI catégoriel** → CLASSIFICATION

== Avantages de cette approche

=== 1. Vraie Compréhension

* L'IA comprend **pourquoi** numérique = régression
* Peut expliquer le raisonnement de façon pédagogique
* Adapte ses explications au dataset spécifique

=== 2. Flexibilité

* Gère les cas ambigus de façon intelligente
* Peut nuancer selon le contexte
* Maintient la capacité d'analyse pour cas complexes

=== 3. Pédagogie

* Explique les concepts ML aux utilisateurs novices
* Donne des exemples concrets avec les vraies données
* Justifie chaque recommandation

== Structure technique

=== Fichiers modifiés

. `ml-pipeline-service/app/ai/llm_service.py`
   - `_get_system_prompt()` : Règles fondamentales renforcées
   - `_build_analysis_prompt()` : Prompt exhaustif avec étapes obligatoires
   - Suppression des court-circuits programmatiques

=== Messages de log

[source,text]
----
🔍 ANALYSE IA: Colonne 'Addiction_Level' de type 'numerical' - L'IA va analyser
----

=== Température et paramètres

* **Température** : 0.3 (cohérence maximale)
* **Max tokens** : Suffisant pour réponses détaillées
* **Format** : JSON obligatoire pour parsing fiable

== Tests et validation

=== Datasets de test recommandés

. **Teen Phone Addiction** : `Addiction_Level` (numerical) → Doit recommander Régression
. **Iris** : `Species` (categorical) → Doit recommander Classification
. **Boston Housing** : `Price` (float) → Doit recommander Régression

=== Critères de succès

* ✅ Recommandation correcte basée sur le type de données
* ✅ Explications spécifiques au dataset
* ✅ Exemples concrets utilisant les vraies données
* ✅ Raisonnement étape par étape visible

== Évolutions futures

=== Améliorations possibles

. Détection automatique des entiers qui sont en réalité continus
. Gestion des colonnes ordinales (catégories ordonnées)
. Interface utilisateur montrant le raisonnement de l'IA
. Feedback utilisateur pour améliorer les prompts

=== Métriques de performance

* **Précision** : % de recommandations correctes sur types évidents
* **Cohérence** : Même résultat pour même input
* **Pédagogie** : Qualité des explications utilisateur

Cette approche garantit que l'IA **comprend vraiment** les concepts ML fondamentaux et peut les expliquer de façon adaptée à chaque dataset spécifique.
