== Vue d'ensemble du Workflow ML → Dataset → XAI

Cette documentation décrit le workflow complet d'intégration entre le Machine Learning, les Datasets et l'eXplainable AI (XAI) dans IBIS-X.

== Architecture du Système

```
Frontend Angular  →  API Gateway  →  ML Pipeline  →  XAI Engine
     ↓                    ↓              ↓             ↓
Dataset Selection    Authentification  Entraînement   Explications
     ↓                    ↓              ↓             ↓
   Columns           User Context    Results + ID    GPT-5 Nano
```

=== 3. Chargement des Colonnes Dataset

[source,typescript]
----
// Frontend: Récupération dataset avec colonnes
async loadDatasetColumns(datasetId: string) {
  const dataset = await this.datasetService.getDataset(datasetId);
  this.datasetColumns = dataset.columns || [];
  
  // ✅ Validation: Colonnes disponibles pour XAI
  if (this.datasetColumns.length > 0) {
    this.canRequestXAIExplanation = true;
  }
}
----

=== 4. Requête XAI avec Contexte Complet

[source,typescript]
----
// Frontend: Requête explicabilité
const xaiRequest = {
  experiment_id: this.experiment.id,
  dataset_id: this.experiment.dataset_id,  // ✅ Dataset ID présent
  explanation_type: "feature_importance",
  audience_level: "technical",
  feature_names: this.datasetColumns.map(col => col.name)
};

await this.xaiService.requestExplanation(xaiRequest);
----

=== 5. Traitement XAI avec GPT-5 Nano

[source,python]
----
# xai-engine-service/app/xai/llm_service.py
class LLMService:
    def __init__(self):
        self.model = "gpt-5-mini"  # ✅ Modèle optimisé performance/coût
        
    async def generate_explanation(self, context: ExplanationContext):
        # ✅ Contexte riche avec colonnes dataset
        instructions = f"""
        Dataset: {context.dataset_id}
        Columns: {context.feature_names}
        Task: {context.task_type}
        Algorithm: {context.algorithm}
        
        Explique les résultats de manière {context.audience_level}.
        """
        
        response = await self.client.responses.create(
            model=self.model,
            instructions=instructions,
            input=context.data,
            reasoning={"effort": "low"}
        )
        return response.content
----

== Points Critiques de l'Intégration

=== ✅ Dataset ID Propagation

Le `dataset_id` doit être présent à chaque étape :

1. **ML Pipeline**: Retourné dans `ExperimentStatus` et `ExperimentResults`
2. **Frontend**: Utilisé pour charger les colonnes dataset  
3. **XAI**: Reçu pour contextualiser les explications

=== ✅ Authentification Utilisateur

```python
# API Gateway → XAI Engine
headers = {
    "X-User-ID": current_user.id,
    "X-User-Email": current_user.email
}

# XAI Engine: Lecture headers
@router.post("/")
async def create_explanation_request(
    x_user_id: str = Header(None, alias="X-User-ID"),
    x_user_email: str = Header(None, alias="X-User-Email")
):
    if not x_user_id:
        raise HTTPException(401, "Authentification requise")
```

=== ✅ Gestion d'État Frontend


== Résolution des Problèmes Courants

=== Dataset ID Manquant

**Symptôme**: `❌ Pas de dataset_id dans le status`

**Solution**: Vérifier que `ExperimentStatus` et `ExperimentResults` incluent `dataset_id`

=== Colonnes Vides

**Symptôme**: `✅ Colonnes chargées: 0 []`

**Solution**: S'assurer que le `dataset_id` est correct et que `datasetService.getDataset()` fonctionne

=== XAI 404/401

**Symptôme**: Erreurs d'authentification ou routing

**Solution**: Vérifier les headers `X-User-ID` et la configuration des routes API Gateway

== Maintenance et Évolutions

=== Ajout d'Algorithmes ML

1. Mettre à jour `schemas.py` pour nouveaux algorithmes
2. Configurer explications spécifiques dans XAI Engine
3. Tester intégration complète

=== Nouveaux Types d'Explications  

1. Étendre `ExplanationType` dans models XAI
2. Implémenter génération dans LLM Service
3. Mettre à jour interface utilisateur

Cette intégration garantit un workflow robuste et maintenable entre tous les composants du système d'explicabilité.
