= Git workflow
:description: Branches, commits, rebase

== Branches

* `main` — toujours déployable. Aucun commit direct.
* `feat/<slug>`, `fix/<slug>`, `docs/<slug>`, `refactor/<slug>`, `chore/<slug>`, `perf/<slug>`, `test/<slug>`, `i18n/<slug>`, `ci/<slug>`.
* Une branche = un objectif. Pas de branche "divers".

== Cycle de travail

[source,bash]
----
git checkout main && git pull
git checkout -b feat/xai-shap-cache

# ... travail + commits ...

git fetch origin
git rebase origin/main     # avant push, garder l'historique linéaire
git push -u origin feat/xai-shap-cache
----

== Conventional Commits

Format : `type(scope): description`

[cols="1,3"]
|===
| Type | Quand
| `feat` | Nouvelle fonctionnalité utilisateur
| `fix` | Correction de bug
| `docs` | Documentation uniquement
| `refactor` | Réorganisation sans changement de comportement
| `test` | Ajout / correction de tests
| `chore` | Tâche de maintenance (deps, config)
| `perf` | Amélioration de performance
| `ci` | CI/CD GitHub Actions
| `i18n` | Traductions FR/EN
| `style` | Formatage (rare, automatisable)
|===

**Scope** : service ou domaine (`api-gateway`, `service-selection`, `ml-pipeline`, `xai-engine`, `frontend`, `k8s`, `docs`).

**Exemples** :

```
feat(ml-pipeline): add LightGBM algorithm option
fix(xai-engine): prevent SHAP crash on single-class datasets
docs(audit): add RGPD compliance checklist
i18n(frontend): translate ML pipeline hero
fix(ml-pipeline): FINDING-014 — remove DEBUG banner
```

== Rebase vs merge

* *Préférer rebase* pour les branches de feature privées (avant PR).
* *Merge commit accepté* sur `main` (GitHub "Squash & merge" est l'option par défaut — l'historique sur `main` reste propre).
* *Jamais* de rebase sur `main` (histoire partagée).

== Commits propres

* Un commit = un changement atomique (principe "green commit").
* Messages à l'impératif présent : "Add X", "Fix Y", pas "Added" / "Fixing".
* Corps du commit optionnel, utile si le *pourquoi* est non évident.

== `.gitignore` discipline

Jamais committer :

* `.env`, `.env.local`, `*.pem`, `kaggle.json`.
* Fichiers générés : `node_modules/`, `__pycache__/`, `htmlcov/`, `dist/`, `build/`.
* Fichiers IDE : `.vscode/` (sauf `settings.json` partagés), `.idea/`.

En cas de secret committé par erreur : révoquer, faire une rotation, coordonner un rebase de correction avec les mainteneurs.
