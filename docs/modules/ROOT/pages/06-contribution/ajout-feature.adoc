= Ajouter une feature
:description: Règle Doc = Code et parcours complet

Règle absolue : **aucune feature n'est complète sans sa documentation**.

== Parcours type (petite feature)

. Issue GitHub décrivant le besoin + critères d'acceptation.
. Branche `feat/<slug>`.
. Écriture du test (TDD recommandé — test qui échoue d'abord).
. Implémentation.
. Documentation dans le pilier adéquat.
. Mise à jour `CHANGELOG.md`.
. PR + revue + merge.

Voir xref:05-audit/qualite/plan-tests.adoc[Plan de tests] :

== Traçabilité

Ajouter une ligne dans xref:05-audit/tracabilite/exigences-implementation.adoc[Matrice exigences ↔ code].

== Invariants à ne jamais casser

* 3 queues Celery (`ml_queue`, `ai_queue`, `xai_queue`).
* `artifact_uri` (pas `model_uri`).
* `users.is_superuser` = propriété calculée (pas de colonne).
* Une `version_table` Alembic par service.
* Angular Material (pas de Tailwind).
* i18n FR+EN toujours synchronisé.
