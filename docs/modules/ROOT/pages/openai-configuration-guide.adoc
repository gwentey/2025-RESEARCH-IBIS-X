== 🎯 Vue d'ensemble

Ce guide vous explique comment configurer l'intégration OpenAI pour le module XAI Engine d'IBIS-X, permettant de générer des explications textuelles intelligentes et adaptées au niveau de chaque utilisateur.

== 📋 Todo Liste Configuration OpenAI

=== ✅ Étape 1 : Créer une Clé API OpenAI

==== 1.1 Créer un Compte OpenAI Platform

. Allez sur https://platform.openai.com[OpenAI Platform^]
. Cliquez sur **"Sign up"** ou **"Log in"** si vous avez déjà un compte
. Vérifiez votre email et complétez l'inscription

==== 1.2 Configurer les Crédits (IMPORTANT)

⚠️ **OpenAI n'est plus gratuit** - vous devez ajouter une méthode de paiement :

. Dans le dashboard OpenAI, allez dans **"Settings" → "Billing"**
. Cliquez **"Add payment method"**
. Ajoutez votre carte de crédit/débit
. **Recommandé** : Définissez une limite mensuelle (ex: $20/mois) dans **"Usage limits"**

==== 1.3 Créer une Clé API

. Dans le dashboard, allez dans **"API Keys"** (section de gauche)
. Cliquez **"+ Create new secret key"**
. Nom suggéré : `IBIS-X-XAI-Engine-Production` ou `IBIS-X-XAI-Engine-Dev`
. **COPIEZ IMMÉDIATEMENT** la clé (elle ne sera plus affichée)
. Format : `sk-proj-ABC123...` (environ 50-60 caractères)

=== ✅ Étape 2 : Configuration Variables d'Environnement

==== 2.1 Mettre à Jour le Fichier .env Local

Ajoutez ces variables dans votre fichier `.env` :

[source,bash]
----
# === OPENAI CONFIGURATION ===
OPENAI_API_KEY=sk-proj-VOTRE_CLE_ICI_ABC123DEF456GHI789...
OPENAI_MODEL=gpt-4o-mini
OPENAI_MAX_TOKENS=2000
OPENAI_TEMPERATURE=0.7
----

⚡ **Modèle Recommandé** : `gpt-4o-mini`
- **Plus intelligent** que gpt-3.5-turbo
- **Moins cher** que gpt-4
- **Optimal** pour les explications XAI

==== 2.2 Variables Complètes Disponibles

[source,bash]
----
# Configuration OpenAI Complète (optionnelles)
OPENAI_API_KEY=sk-proj-votre_cle           # OBLIGATOIRE
OPENAI_MODEL=gpt-4o-mini                   # Modèle LLM (défaut: gpt-3.5-turbo)
OPENAI_MAX_TOKENS=2000                     # Longueur max réponse (défaut: 1000)  
OPENAI_TEMPERATURE=0.7                     # Créativité 0-1 (défaut: 0.7)
MAX_CHAT_QUESTIONS=5                       # Questions max par session (défaut: 5)
CHAT_SESSION_TIMEOUT_HOURS=24              # Timeout session chat (défaut: 24h)
MAX_TOKENS_PER_QUESTION=200                # Tokens max par question (défaut: 200)
----

=== ✅ Étape 3 : Déploiement avec OpenAI

==== 3.1 Redéployer IBIS-X

[source,bash]

- Créer/mettre à jour `k8s/base/xai-engine/secrets.yaml`

== 🔐 Sécurité et Bonnes Pratiques

=== Protection de la Clé API

⚠️ **CRITIQUE** : Ne jamais exposer votre clé OpenAI

**✅ Bonne pratique :**
[source,bash]
----
# Dans .env (non versionné)
OPENAI_API_KEY=sk-proj-...

# Dans .gitignore
.env
.env.*
----

**❌ JAMAIS :**
[source,bash]
----
# Dans le code source
OPENAI_API_KEY = "sk-proj-abc123..."  # ❌ DANGER

# Dans les logs
logger.info(f"Using key: {api_key}")  # ❌ EXPOSITION
----

=== Rotation des Clés

. **Créez une nouvelle clé** tous les 3-6 mois
. **Testez la nouvelle clé** en développement
. **Mettez à jour** `.env` et redéployez
. **Révoquez l'ancienne clé** sur OpenAI Platform

== 🐛 Dépannage

=== Erreurs Communes

**"Invalid API Key"**
[source,bash]
----
# Vérifier la clé dans les secrets
kubectl get secret xai-engine-secrets -n ibis-x -o yaml

# Décoder la clé
echo "ENCODED_KEY" | base64 -d
----

**"Rate limit exceeded"**
- Augmentez les limites sur OpenAI Platform
- Implémentez un retry avec backoff exponential
- Considérez un modèle moins cher

**"Model not found"**
- Vérifiez que `OPENAI_MODEL` existe
- Models disponibles : gpt-3.5-turbo, gpt-4o-mini, gpt-4

=== Logs de Debug
