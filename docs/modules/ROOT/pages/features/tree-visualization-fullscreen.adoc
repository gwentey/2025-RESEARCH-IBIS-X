== Vue d'ensemble

La fonctionnalité de visualisation fullscreen permet aux utilisateurs d'examiner en détail l'arbre de décision généré lors de l'entraînement d'un modèle. Cette fonctionnalité est particulièrement utile pour les arbres complexes qui nécessitent plus d'espace d'affichage.

== Caractéristiques principales

=== Interface utilisateur

* **Bouton Fullscreen** : Un bouton flottant violet/vert avec l'icône fullscreen, positionné en haut à droite de la carte de visualisation d'arbre
* **Popup Fullscreen** : Interface dédiée avec fond sombre pour une meilleure concentration
* **Contrôles intuitifs** : Fermeture via bouton, touche Échap ou clic sur l'overlay

== Architecture technique

=== Approche JavaScript pur

La solution utilise JavaScript pur pour contourner les problèmes d'overlay Angular :

[source,typescript]
----
private createFullscreenButton(): void {
  // Création du bouton avec JavaScript pur
  const fullscreenBtn = document.createElement('div');
  fullscreenBtn.id = 'tree-fullscreen-btn';
  
  // Event listener JavaScript pur avec contexte préservé
  const self = this;
  buttonElement.addEventListener('click', (event) => {
    event.preventDefault();
    event.stopPropagation();
    event.stopImmediatePropagation();
    self.openTreeFullscreenPopup();
  });
}
----

=== Gestion du contexte Angular

Pour préserver le contexte du composant Angular dans les event listeners JavaScript :

[source,typescript]
----
// Capture explicite du contexte
const self = this;

// Utilisation dans l'event listener
buttonElement.addEventListener('click', (event) => {
  // Utiliser 'self' au lieu de 'this'
  self.openTreeFullscreenPopup();
});
----

=== Clone de l'arbre

L'arbre est cloné dans la popup pour préserver l'original :

[source,typescript]
----
private loadTreeInFullscreen(): void {
  const originalTree = document.querySelector('app-real-tree-visualization');
  const treeClone = originalTree.cloneNode(true) as HTMLElement;
  container.appendChild(treeClone);
}
----

== Problèmes résolus

=== Overlay invisible

**Problème** : Un overlay invisible capturait tous les clics, empêchant l'interaction avec les boutons.

**Solution** : 
- Utilisation de JavaScript pur pour créer et gérer les event listeners
- Z-index maximum (2147483647) pour garantir la visibilité
- `stopImmediatePropagation()` pour empêcher la propagation d'événements

=== Perte de contexte

**Problème** : Le contexte `this` était perdu lors du clonage d'éléments DOM.

**Solution** : Capture explicite du contexte avec `const self = this` avant la création des event listeners.

== Utilisation

=== Pour les utilisateurs

1. **Ouvrir le fullscreen** : Cliquer sur le bouton flottant violet/vert en haut à droite de la visualisation d'arbre
2. **Naviguer** : Utiliser la souris pour explorer l'arbre en détail
3. **Fermer** : 
   - Cliquer sur le bouton de fermeture (X rouge)
   - Appuyer sur la touche Échap
   - Cliquer en dehors de la zone de contenu

=== Pour les développeurs

[source,typescript]
----
// Le setup est automatique lors du chargement du composant
ngAfterViewInit(): void {
  this.initializeFullscreenSystem();
}

// Initialisation du système fullscreen
private initializeFullscreenSystem(): void {
  setTimeout(() => {
    this.setupPureJavaScriptListeners();
  }, 500);
}
----

== Performances

* **Clonage DOM** : L'arbre est cloné plutôt que déplacé pour éviter les problèmes de state Angular
* **Event delegation** : Utilisation minimale d'event listeners pour optimiser les performances
* **Lazy loading** : L'arbre n'est chargé dans la popup qu'après son ouverture

== Accessibilité

* **Navigation clavier** : Support de la touche Échap pour fermer
* **Indicateurs visuels** : Effets de hover sur le bouton fullscreen
* **Instructions claires** : Message indiquant comment fermer la popup

== Maintenance

=== Points d'attention

1. **Z-index conflicts** : Le z-index maximum peut entrer en conflit avec d'autres éléments
2. **Memory leaks** : S'assurer que les event listeners sont correctement supprimés
3. **Compatibilité** : Tester sur différents navigateurs pour la compatibilité CSS

=== Amélioration futures

* Support du zoom avec molette de souris
* Export de l'arbre en image haute résolution
* Mode présentation avec navigation guidée
* Support tactile pour tablettes

== Conclusion

La fonctionnalité de visualisation fullscreen améliore significativement l'expérience utilisateur pour l'analyse des arbres de décision complexes. L'approche JavaScript pur garantit une compatibilité maximale et contourne efficacement les limitations du framework Angular.
