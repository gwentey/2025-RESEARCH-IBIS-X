= Visualisation Interactive de l'Arbre de Décision
:page-tags: ml, visualization, decision-tree, interactive
:page-description: Interface interactive pour visualiser et explorer les arbres de décision avec tableau blanc

== Vue d'ensemble

L'interface de visualisation interactive des arbres de décision permet d'explorer en détail la structure des modèles Decision Tree et Random Forest dans un environnement de tableau blanc interactif.

== Fonctionnalités principales

=== Affichage de l'arbre

L'arbre est affiché avec :

* **Nœuds colorés** : 
  - Bleu pour les nœuds de décision
  - Vert pour les feuilles
* **Informations détaillées** : Chaque nœud affiche la condition, le nombre d'échantillons et la valeur
* **Liens visuels** : Connexions entre parent et enfants

=== Contrôles interactifs

[cols="1,3"]
|===
|Contrôle |Description

|**Zoom**
|Molette de souris ou boutons +/- pour zoomer

|**Pan/Déplacement**
|Cliquer-glisser sur le fond pour déplacer la vue

|**Déplacement des nœuds**
|Cliquer-glisser sur un nœud pour le repositionner

|**Espacement**
|Boutons pour ajuster l'espacement horizontal entre nœuds

|**Réinitialisation**
|Bouton pour réorganiser l'arbre dans sa disposition originale
|===

=== Raccourcis clavier

* `R` : Réinitialiser la vue
* `+/-` : Zoom avant/arrière
* `WASD` ou flèches : Déplacer la vue
* `Escape` : Fermer le mode plein écran

== Utilisation

=== Ouvrir la visualisation

1. Depuis la page de résultats d'entraînement
2. Cliquer sur le bouton fullscreen (🗖) sur la carte de visualisation de l'arbre
3. Le tableau blanc interactif s'ouvre en plein écran

=== Explorer l'arbre

[source,text]
----
1. Utilisez la molette pour zoomer sur des parties spécifiques
2. Cliquez et glissez le fond pour naviguer
3. Réorganisez les nœuds en les déplaçant individuellement
4. Ajustez l'espacement pour mieux voir la structure
----

=== Exporter l'arbre

Le bouton d'export permet de sauvegarder l'arbre au format SVG pour une utilisation ultérieure.

== Architecture technique

=== Frontend

L'implémentation utilise :

* **HTML/CSS natif** pour les nœuds (draggables)
* **SVG** pour les liens entre nœuds
* **JavaScript pur** pour les interactions (pas de librairie externe)

[source,typescript]
----
// Structure des données d'un nœud
interface TreeNode {
  id: string;
  name: string;
  condition: string;
  samples: number;
  isLeaf: boolean;
  x: number;
  y: number;
  width: number;
  height: number;
}
----

=== Backend

Le backend génère la structure de l'arbre :

[source,python]
----
# Dans evaluation.py
if hasattr(model, 'get_tree_structure'):
    tree_structure = model.get_tree_structure(
        feature_names=feature_names,
        class_names=class_names
    )
    visualizations['tree_structure'] = tree_structure
----

=== Format des données

Les données de l'arbre sont transmises en JSON :

[source,json]
----
{
  "name": "feature_name",
  "condition": "≤ 0.5",
  "samples": 100,
  "is_leaf": false,
  "feature": "feature_name",
  "threshold": 0.5,
  "children": [
    {
      "name": "Classe A",
      "condition": "n=60",
      "samples": 60,
      "is_leaf": true,
      "class_name": "Classe A"
    }
  ]
}
----

== Cas d'usage

=== Analyse de la structure du modèle

* Comprendre comment le modèle prend ses décisions
* Identifier les features les plus discriminantes
* Détecter les branches profondes ou déséquilibrées

=== Présentation et documentation

* Exporter des visualisations pour des rapports
* Créer des supports pédagogiques
* Documenter le comportement du modèle

=== Débogage et optimisation

* Identifier les nœuds avec peu d'échantillons
* Repérer les branches trop complexes
* Analyser la distribution des classes dans l'arbre

== Limitations et considérations

* **Performance** : Pour des arbres très larges (>1000 nœuds), les performances peuvent être dégradées
* **Espace** : Les arbres très profonds peuvent nécessiter beaucoup de zoom/pan
* **Random Forest** : Seul un arbre représentatif est affiché, pas toute la forêt

== Évolutions futures
