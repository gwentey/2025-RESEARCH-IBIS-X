= Correction Synchronisation Scores Éthiques Heatmap
:description: Solution pour corriger l'incohérence entre les scores éthiques affichés dans la heatmap
:keywords: heatmap, éthique, synchronisation, ASAP Essay, datasets
:page-layout: admin

== Problème Identifié

Le dataset **ASAP Essay Scoring** présentait une incohérence dans l'affichage des scores éthiques :

* *Fichier JSON enrichi* : 45% d'éthique
* *Interface heatmap* : 100% d'éthique  
* *Score attendu* : ~50% d'éthique

Cette incohérence provenait d'un problème de synchronisation entre les métadonnées enrichies et les templates par défaut utilisés par l'API.

== Cause Racine

=== Templates Par Défaut Optimistes

Le système utilise des templates par défaut pour le domaine "education" qui assignent des valeurs éthiques élevées :

[source,python]
----
{
    "domain": "education",
    "ethical": {
        "informed_consent": True,
        "transparency": True,
        "user_control": False,  // Seul critère à False
        "equity_non_discrimination": True,
        "security_measures_in_place": True,
        "data_quality_documented": True,
        "anonymization_applied": True,
        "record_keeping_policy_exists": True,
        "purpose_limitation_respected": True,
        "accountability_defined": True
    }
}
----

*Résultat* : 9/10 critères = **90% d'éthique**

=== Métadonnées JSON Réalistes

Le fichier JSON enrichi contenait des valeurs plus réalistes basées sur l'analyse du dataset :

[source,json]
----
{
  "quality_metrics": {
    "ethical_compliance_score": 45
  }
}
----

== Solution Implémentée

=== Étape 1: Correction Métadonnées JSON

Mise à jour du fichier `datasets/kaggle-import/enriched_metadata/asap_essay_scoring.json` avec des valeurs réalistes :

[source,json]
----
{
  "ethical_compliance": {
    "informed_consent": false,        // ❌ Pas de consentement explicite
    "transparency": true,             // ✅ Processus transparent
    "user_control": false,            // ❌ Pas de contrôle étudiant
    "equity_non_discrimination": true, // ✅ Évaluation équitable
    "security_measures": false,        // ❌ Pas de sécurité documentée
    "data_quality_documented": true,   // ✅ Qualité documentée
    "anonymization_applied": true,     // ✅ Anonymisation appliquée
    "record_keeping_policy": false,    // ❌ Pas de politique conservation
    "purpose_limitation": false,       // ❌ Usage non limité
    "accountability_defined": true     // ✅ Responsabilité définie
  },
  "quality_metrics": {
    "completeness_score": 100,
    "consistency_score": 75,
    "ethical_compliance_score": 50   // 5/10 critères = 50%
  }
}
----

=== Étape 2: Scripts de Correction

Création de scripts automatisés :

* `scripts/sync_asap_essay_metadata.py` : Met à jour le fichier JSON
* `service-selection/scripts/fix_asap_ethics_sync.py` : Corrige la base de données
* `scripts/debug_heatmap_data.py` : Debug les données de la heatmap

=== Étape 3: Validation

*Avant correction* :
- JSON : 45% éthique
- Template BDD : 90% éthique  
- Interface : 100% éthique

*Après correction* :
- JSON : 50% éthique (5/10 critères)
- Template BDD : 50% éthique (synchronized)
- Interface : 50% éthique (cohérent)

== Caractéristiques Éthiques du Dataset ASAP Essay

Le dataset **ASAP - Essay Scoring** présente les caractéristiques éthiques suivantes :

[cols="1,1,3"]

|`purpose_limitation` |❌ False |Usage potentiel au-delà de l'évaluation initiale

|===

*Score final* : **5/10 = 50%** ✅
