= Référence Technique XAI Engine

Cette documentation technique détaille l'architecture, les API et l'implémentation du moteur d'explicabilité XAI d'IBIS-X.

== 🏗️ Architecture du système

=== Vue d'ensemble

Le XAI Engine est un microservice indépendant qui s'intègre dans l'écosystème IBIS-X :

[source,mermaid]
----
graph TB
    Frontend[Frontend Angular] --> Gateway[API Gateway]
    Gateway --> XAI[XAI Engine Service]
    XAI --> Redis[Redis/Celery]
    XAI --> DB[(PostgreSQL)]
    XAI --> Storage[MinIO/Azure Storage]
    XAI --> OpenAI[OpenAI API]
    
    Redis --> Workers[XAI Celery Workers]
    Workers --> ML[Modèles ML]
    Workers --> Explainers[SHAP/LIME]
----

=== Composants principaux

==== XAI Engine Service (FastAPI)
* **Responsabilité** : API REST pour les demandes d'explication
* **Port** : 8083
* **Technologies** : FastAPI, SQLAlchemy, Pydantic

==== XAI Celery Workers
* **Responsabilité** : Traitement asynchrone des explications
* **Queue** : `xai_queue`
* **Technologies** : Celery, SHAP, LIME, scikit-learn

==== Base de données
* **Tables principales** : `explanation_requests`, `chat_sessions`, `chat_messages`, `explanation_artifacts`
* **Relations** : Liens avec `experiments` (ML Pipeline) et `users` (API Gateway)

== 🧠 Algorithmes d'explicabilité

=== SHAP (SHapley Additive exPlanations)

==== Principe
SHAP utilise la théorie des jeux pour attribuer équitablement l'importance de chaque feature dans une prédiction.

==== Implémentation
[source,python]
----
import shap
from sklearn.ensemble import RandomForestClassifier

def generate_shap_explanation(model, X_train, X_test, feature_names):
    if isinstance(model, (RandomForestClassifier, DecisionTreeClassifier)):
        # TreeExplainer pour les modèles basés sur les arbres
        explainer = shap.TreeExplainer(model)
    else:
        # KernelExplainer pour les autres modèles
        explainer = shap.KernelExplainer(model.predict_proba, X_train.sample(100))
    
    shap_values = explainer.shap_values(X_test)
    
    # Génération des visualisations
    shap.summary_plot(shap_values, X_test, feature_names=feature_names, show=False)
    plt.savefig('shap_summary.png', bbox_inches='tight', dpi=300)
    
    return {
        "shap_values": shap_values,
        "base_values": explainer.expected_value,
        "feature_importance": dict(zip(feature_names, shap_values.mean(0)))
    }
----

==== Avantages
* Garanties théoriques solides
* Très efficace pour les modèles en arbres
* Visualisations riches et interprétables

==== Limitations
* Peut être lent sur de gros datasets
* Nécessite des données d'entraînement pour certains explainers

=== LIME (Local Interpretable Model-agnostic Explanations)

==== Principe
LIME explique localement n'importe quel modèle en ajustant un modèle simple (linéaire) autour de l'instance à expliquer.

==== Implémentation
[source,python]
----
import lime
import lime.lime_tabular

def generate_lime_explanation(model, X_train, X_test, feature_names, instance_idx=0):
    explainer = lime.lime_tabular.LimeTabularExplainer(
        X_train,
        feature_names=feature_names,
        class_names=['Class 0', 'Class 1'],
        mode='classification'
    )
    
    explanation = explainer.explain_instance(
        X_test[instance_idx], 
        model.predict_proba,
        num_features=len(feature_names)
    )
    
    return {
        "explanation": explanation,
        "feature_importance": dict(explanation.as_list()),
        "score": explanation.score
    }
----

==== Avantages
* Agnostic au modèle (fonctionne avec n'importe quel ML)
* Explications locales très précises
* Implémentation simple et rapide

==== Limitations
* Seulement local (pas d'explication globale)
* Sensible au choix des perturbations
* Peut être instable sur des données bruitées

=== Sélection automatique de l'algorithme

[source,python]
----
def select_best_explainer(model, dataset_size, model_type):
    """Sélectionne automatiquement le meilleur explainer selon le contexte."""
    
    if model_type in ['RandomForest', 'DecisionTree', 'GradientBoosting']:
        if dataset_size < 10000:
            return 'SHAP_Tree'  # Optimal pour les arbres sur petites données
        else:
            return 'SHAP_Sampling'  # SHAP avec échantillonnage
    
    elif model_type in ['LogisticRegression', 'SVM']:
        return 'SHAP_Linear'  # Explainer linéaire optimisé
    
    else:
        if dataset_size < 5000:
            return 'LIME'  # LIME pour les petits datasets, modèles complexes
        else:
            return 'SHAP_Kernel'  # SHAP généraliste avec échantillonnage
----

def generate_contextual_questions(explanation_results, user_profile, model_performance):
    """Génère des questions pertinentes selon le contexte."""
    
    questions = []
    
    # Questions basées sur l'importance des features
    top_features = get_top_features(explanation_results, n=3)
    questions.extend([
        f"Pourquoi la variable '{feature}' est-elle si importante ?",
        f"Comment interpréter l'impact de '{feature}' sur les prédictions ?"
        for feature in top_features
    ])
    
    # Questions selon les performances
    if model_performance['accuracy'] < 0.8:
        questions.append("Comment améliorer la performance de mon modèle ?")
    
    if detect_potential_bias(explanation_results):
        questions.append("Y a-t-il des biais dans mes prédictions ?")
    
    # Questions selon le profil utilisateur
    if user_profile == 'business':
        questions.append("Que signifient ces résultats pour mon business ?")
    elif user_profile == 'technical':
        questions.append("Quels sont les hyperparamètres à optimiser ?")
    
    return questions[:5]  # Maximum 5 questions
----

=== Parallélisation des calculs

[source,python]

from concurrent.futures import ThreadPoolExecutor

def parallel_explanation_generation(model, X_test, methods=['SHAP', 'LIME']):
    """Génère plusieurs explications en parallèle."""
    
    with ThreadPoolExecutor(max_workers=2) as executor:
        futures = []
        
        if 'SHAP' in methods:
            futures.append(executor.submit(generate_shap_explanation, model, X_test))
        
        if 'LIME' in methods:
            futures.append(executor.submit(generate_lime_explanation, model, X_test))
        
        results = [future.result() for future in futures]
    
    return combine_explanations(results)
----

def smart_sampling_for_explanation(X_data, max_samples=1000):
    """Échantillonne intelligemment les données pour l'explication."""
    
    if len(X_data) <= max_samples:
        return X_data
    
    # Stratégies d'échantillonnage
    strategies = {
        'random': X_data.sample(max_samples),
        'stratified': stratified_sample(X_data, max_samples),
        'representative': get_representative_sample(X_data, max_samples)
    }
