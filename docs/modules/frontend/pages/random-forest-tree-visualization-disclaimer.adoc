= Disclaimer pour Visualisation d'Arbre Random Forest
:navtitle: Disclaimer Arbre RF
:description: Ajout d'un disclaimer pour éviter la confusion lors de l'affichage d'un arbre individuel dans Random Forest

== Problématique

=== Risque de Confusion

Lors de l'affichage d'un arbre de décision pour un modèle **Random Forest**, il existe un risque significatif de **mauvaise interprétation** :

[cols="1,2"]
|===
|**Problème** |**Impact**

|Un seul arbre affiché
|L'utilisateur pense que cet arbre représente toute la logique du modèle

|Logique aléatoire
|Chaque arbre est entraîné sur des données et variables aléatoires

|Décision collective ignorée
|Random Forest combine 100+ arbres, pas un seul

|Surajustement individuel
|Un arbre seul peut montrer du surajustement que la forêt corrige
|===

=== Spécificités Random Forest

Random Forest utilise plusieurs techniques aléatoires :

1. **Bootstrap Sampling** : Chaque arbre est entraîné sur un échantillon aléatoire des données
2. **Random Feature Selection** : À chaque division, seules quelques variables sont considérées aléatoirement
3. **Voting/Averaging** : La décision finale combine tous les arbres (vote majoritaire ou moyenne)

**➜ Un arbre individuel ne représente donc PAS la logique complète du modèle.**

== Solution Implémentée

=== Disclaimer Visible

Un avertissement clair et informatif a été ajouté **uniquement pour Random Forest** :

[source,html]
----
<div *ngIf="getAlgorithm() === 'random_forest'" class="tree-disclaimer">
  <div class="disclaimer-icon">
    <mat-icon>info</mat-icon>
  </div>
  <div class="disclaimer-content">
    <h4>⚠️ Arbre d'Exemple (Random Forest)</h4>
    <p>
      <strong>Cet arbre est un exemple parmi {{getTreeCount()}}.</strong> 
      Il ne représente pas la logique complète de Random Forest, mais illustre 
      comment un arbre individuel prend ses décisions.
    </p>
    <!-- Détails techniques -->
  </div>
</div>
----

=== Contenu du Disclaimer

Le disclaimer inclut :

1. **Message principal** : "Cet arbre est un exemple parmi 100"
2. **Clarification** : "Il ne représente pas la logique complète"
3. **Utilité pédagogique** : "mais illustre comment un arbre individuel fonctionne"
4. **Détails techniques** :
   * Données aléatoires (bootstrap sampling)
   * Variables aléatoires à chaque division  
   * Décision finale = vote de 100 arbres

=== Design du Disclaimer

**Couleurs et Style :**
* Fond orange clair pour attirer l'attention sans alarmer
* Bordure gauche orange pour l'importance
* Icône d'information circulaire
* Texte structuré et lisible

**Positionnement :**
* Affiché **avant** la visualisation de l'arbre
* Impossible à manquer
* Responsive design pour mobile
