== Vue d'ensemble

L'intégration WebSHAP dans IBIS-X apporte de vraies valeurs SHAP interactives, remplaçant les graphiques d'importance statiques par des explications dynamiques calculées en temps réel dans le navigateur.

WARNING: Cette intégration utilise une implémentation personnalisée qui simule WebSHAP sans la librairie npm officielle en raison de problèmes d'installation dans l'environnement Docker.

=== Structure de Données SHAP

[source,typescript]
----
interface SHAPData {
  feature_names: string[];      // Noms des variables
  feature_values: number[];     // Valeurs des variables pour l'instance
  shap_values: number[];        // Contributions SHAP pour chaque variable
  base_value: number;           // Valeur de référence du modèle
  prediction: number;           // Prédiction finale
}
----

== Fonctionnalités Implémentées

=== 1. Visualisation Interactive SVG

* **Génération dynamique** : Le composant crée des graphiques SVG natifs
* **Barres colorées** : Vert pour contributions positives, rouge pour négatives
* **Animations** : Effets d'entrée progressive pour une meilleure UX
* **Interactivité** : Hover effects et tooltips informatifs

=== 2. Calcul des Valeurs SHAP

[source,typescript]
----
private async computeSHAPValues(
  instanceData: number[],
  backgroundData: number[],
  experimentResults: any,
  featureNames: string[]
): Promise<number[]>
----

La méthode simule l'algorithme Kernel SHAP en :

1. Récupérant l'importance des features depuis le modèle existant
2. Calculant l'écart entre l'instance et la baseline
3. Pondérant par l'importance et ajoutant de la variabilité

=== 3. Interface Utilisateur

==== Éléments Visuels

* **Card Material Design** : Intégration avec le design system IBIS-X
* **Zone de visualisation** : Container responsive avec SVG centré
* **Détails techniques** : Informations sur la base value, prédiction, nombre de features
* **Boutons d'action** : Rafraîchissement et export SVG

== Flux de Données

=== 1. Récupération des Données

1. **Résultats d'expérience** : Appel à `mlPipelineService.getExperimentResults()`
2. **Importance des features** : Extraction depuis `experimentResults.feature_importance`
3. **Noms des features** : `Object.keys(featureImportance)`

=== 2. Calcul SHAP

1. **Données de baseline** : Array de zéros (peut être amélioré)
2. **Instance de données** : Données utilisateur ou échantillon généré
3. **Valeurs SHAP** : Simulation basée sur l'importance et la déviation

=== 3. Rendu Visuel

1. **Création SVG** : Génération programmatique des éléments
2. **Échelle** : Calcul automatique basé sur les valeurs min/max
3. **Animation** : Effets progressifs avec CSS transitions

== Dépannage

=== Problèmes Courants

==== Pas de Données SHAP

**Symptôme** : Message "Aucune donnée disponible pour les explications SHAP"

**Causes possibles** :
* L'expérience n'est pas terminée
* Pas d'importance des features dans les résultats
* Problème de réseau

**Solution** : Vérifier que `experimentResults.feature_importance` contient des données

==== Erreur de Rendu SVG

**Symptôme** : Visualisation vide ou erreur JavaScript

**Causes possibles** :
* Données SHAP malformées
* Problème de dimensions SVG
* Erreur dans les calculs d'échelle

**Solution** : Vérifier les logs console et valider les données d'entrée
