= Améliorations Page Résultats d'Expérimentation: Classification + Decision Tree
:navtitle: Résultats Classification DT
:description: Améliorations spécifiques pour la page des résultats d'expérimentation pour le cas Classification avec Decision Tree

== Vue d'ensemble

Cette documentation présente les améliorations apportées à la page des résultats d'expérimentation, spécifiquement pour le cas **Classification + Decision Tree**, suite aux retours d'analyse et critiques d'utilisation.

=== Contexte
L'utilisateur a testé la combinaison Classification + Decision Tree et identifié plusieurs améliorations nécessaires :

* **KPIs** : Mettre F1-macro comme métrique principale au lieu de F1-Score global
* **Graphiques** : Ajouter courbes ROC/PR pour classification binaire
* **Explications** : Ajouter explications locales avec chemin de décision

== Améliorations Implémentées

=== 1. Système de Métriques Amélioré

==== F1-macro comme Métrique Principale
[source,typescript]
----
// Nouvelle méthode pour détection classification binaire/multi-classes
getClassificationType(): 'binary' | 'multiclass' | 'unknown' {
  // Logique de détection basée sur métadonnées et métriques
}

// KPI principal amélioré avec contexte spécifique
getPrimaryKPI(): {key: string, value: number, displayName: string, description: string, isImproved?: boolean}
----

**Points clés :**

* F1-macro prioritaire pour Classification + Decision Tree
* Descriptions adaptées selon le contexte (binaire vs multi-classes)
* Marquage des améliorations (`isImproved` flag)

==== Détection Classification Binaire vs Multi-classes

La page détecte maintenant automatiquement le type de classification :

. **Sources de détection :**
* Métadonnées des résultats (`n_classes`)
* Matrice de confusion
* Présence de métriques ROC-AUC/PR-AUC

. **Impact sur l'affichage :**
* Descriptions adaptées des métriques
* Visualisations spécifiques (ROC/PR curves)
* Conseils d'interprétation contextuels

=== 2. Visualisations Améliorées

==== Support Courbes ROC/PR pour Classification Binaire

Nouvelles méthodes ajoutées :
[source,typescript]
----
shouldShowROCPRCurves(): boolean
hasPRCurve(): boolean  
getPRCurveImage(): string
----

**Fonctionnalités :**

* Détection automatique si ROC/PR curves doivent être affichées
* Support URL pour courbe Precision-Recall
* Messages informatifs pour visualisations manquantes mais attendues

==== Visualisations Priorisées

Les visualisations sont maintenant classées par priorité :

1. **Arbre de décision** (priorité 1)
2. **Matrice de confusion** (priorité 2) 
3. **Courbe ROC** (priorité 3)
4. **Courbe PR** pour binaire (priorité 4)
5. **Importance variables** (priorité 5)

==== Tags Contextuels

* **Classification Binaire** : tag informatif sur courbe ROC
* **Classes déséquilibrées** : tag spécialisé sur courbe PR

=== 3. Section Explications Locales (NOUVEAU)

Une nouvelle section dédiée aux explications locales pour les arbres de décision :

==== Composants de la Section

. **Introduction explicative**
* Description du concept d'explications locales
* Variables disponibles dans le dataset
* Différenciation classification vs régression

. **Exemple de chemin de décision**
* Simulation d'un chemin dans l'arbre
* Visualisation étape par étape
* Résultat final avec confiance

. **Appel à l'action XAI**
* Lien vers analyses plus poussées
* Fonctionnalités disponibles avec SHAP

=== 4. Conseils d'Interprétation Enrichis

==== Conseils Spécifiques par Type

Les conseils d'interprétation s'adaptent maintenant au contexte :

* **Matrice de confusion** : différent pour binaire (2×2) vs multi-classes
* **Courbe ROC** : explication spécifique binaire vs one-vs-rest
* **Courbe PR** : mise en avant pour classes déséquilibrées
* **Explications locales** : nouveau conseil pour arbres de décision
